Transfer Utils | Portal Provider

Easy CFDI Import For Providers

Features.

  • Create or attach CFDI to an invoice from purchase order.
  • Multiple validations to make the creation of invoices safe.
  • Updating the SKU of the products from the portal.
  • Import and export the list of products for mass update.
Configuration

To start to use this module, you have to configure the option in this module.

  • This configuration is used to set the default state of odoo invoice.
Product Configuration

You need to add the vendor and the SKU (product identifier) ​​to the product variant.

Important

The concepts in XML needs the attribute 'NoIdentificacion' and product sku in odoo for invoice validation.

Validations

By default there are the following validations.

The same as the transfer utils module.

  • XML structure.
  • Existence in the SAT.
  • Related UUID.

To search and attach there are the following validations.

  • Existence and validation of the partner.
  • Price and quantity.
  • Total amount of the invoice.
  • That the quantity invoiced and quantity received match the xml to be invoiced.
Create or Attach Multiple CFDI to a Purchase Order Invoices

For attach CFDI to a purchase order invoice there is the option of importing massively through a compressed file or individually with an xml file. This from odoo portal.

For all xml the modal will display the import result.

Update SKUs Products

We added a new page to update the SKUs of the supplier's products. Here you can update each product variant manually or do a mass update using a csv.

For any result of the update, a notification will be displayed on the side of the page.

To export in csv with all the products you just have to click on the button to start the download.

To import it is necessary the same format that is given when exporting the csv. By selecting the file and clicking the import button, the SKUs will automatically be updated.
